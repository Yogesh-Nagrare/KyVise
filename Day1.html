How Does Internet Work

Principle 1: The Need to Share Information

The most fundamental truth is that we have information on one computer (Computer A) and we want it to be on another computer (Computer B). These computers are not in the same room.

The simplest way is to carry the information physically (USB stick), but it’s slow and doesn’t scale. A better way is to run a cable and send signals as 1s and 0s, creating a direct link.

Principle 2: The Problem of Scale

Connecting multiple computers directly requires too many cables. Solution: Use a central device called a Switch. This forms a Local Network.

Principle 3: The Problem of Connecting Networks

To connect different local networks, we use a Router. The Internet is a network of networks connected by routers.

Principle 4: The Problem of Finding Anyone (Addressing)

Every device needs a unique identifier called an IP Address. Routers use IPs to forward packets step by step.

Principle 5: The Problem of Reliability and Fairness

Large data is split into Packets. Each contains source, destination, and order number. Protocols like TCP/IP ensure reliable delivery.

Summary from First Principles

  1. Direct physical link between two computers.
  2. Local network with a switch.
  3. Internetwork with routers.
  4. Unique addressing with IP addresses.
  5. Reliable transfer using TCP/IP packets.

IP Address

IPv4: 32-bit number, about 4.3 billion addresses. Example: 172.217.16.142.
Problem: Limited addresses → led to Public & Private IPs.

Public IP Address

  • Unique global address assigned by ISP.
  • Used by servers (Google, Netflix, etc.).
  • Your router has one public IP.

Private IP Address

  • Used only inside local networks (e.g., 192.168.x.x).
  • Not globally unique.
  • Assigned by router to devices.

IPv6

IPv6: 128-bit address (~340 undecillion addresses).
Written in hexadecimal, separated by colons. Example: 2001:0db8:85a3:0000:0000:8a2e:0370:7334.

MAC Address

A unique 48-bit number burned into hardware. Display formats:

  • Colon-separated: 3C:22:FB:A3:B4:C5
  • Hyphen-separated: 3C-22-FB-A3-B4-C5
  • Period-separated: 3c22.fba3.b4c5

Finding MAC Address

Windows: Use ipconfig /all or getmac /v in Command Prompt.

MacOS: Use ifconfig in Terminal.

Port Number

Ports identify services on a device. Range: 0–65535 (16-bit).

Categories of Ports

  1. Well-Known Ports (0–1023): HTTP (80), HTTPS (443), SSH (22), FTP (21), DNS (53).
  2. Registered Ports (1024–49151): React/Node (3000), MySQL (3306), MongoDB (27017).
  3. Dynamic/Ephemeral Ports (49152–65535): Temporary ports assigned to clients.

DNS (Domain Name System)

DNS translates domain names into IP addresses.

Steps of DNS Resolution

  1. Check local cache: Browser → OS → Router.
  2. Recursive Resolver: ISP or public resolver (Google 8.8.8.8, Cloudflare 1.1.1.1).
  3. Root Servers: Tell where TLD servers are (.com, .org, etc.).
  4. TLD Servers: Direct to authoritative name servers.
  5. Authoritative Servers: Provide the actual IP address.

Once resolved, the browser connects to the IP on the correct port (e.g., 443 for HTTPS).
